# 🛡️ Remaster 公會：任務簽到系統開發規格書 (V2.5)

## 1. 專案概述 (Project Vision)
本專案為「2026 鐵人賽：JavaScript 核心 Remaster」專屬的沉浸式任務管理系統。將開發流程 RPG 化，讓參與者（勇者）在修煉過程中具備更強的歸屬感與社群互動。

- **專案名稱**：Remaster Guild Hub (公會任務簽到中心)
- **核心調性**：現代 RPG 風格 (Fantasy x Sci-Fi)、數據美學、一頁式儀表板。

---

## 2. 技術棧 (Tech Stack)
- **前端框架**：React 18 (Hooks 驅動)
- **樣式處理**：Tailwind CSS (自定義主題、玻璃擬態)
- **圖標庫**：Lucide React
- **頭像生成**：DiceBear API (Avatar Seeds: Felix, Aneka, Caleb, Jocelyn)
- **資料庫（預計）**：Supabase (PostgreSQL + Real-time)

---

## 3. 視覺系統 (Design System)

### 3.1 核心配色 (Palette)
- **底色 (Deep Dark)**：`#07090e` (背景)、`#0b0e14` (導覽列)
- **主色 (Energy Yellow)**：`#FFD43B` (重點、圖章、主按鈕)
- **輔色 (Guild Blue)**：`#4DABF7` (連結、進度條、酒館)
- **表面色 (Glass)**：`#131720` (透明度 80% + Backdrop Blur)

### 3.2 UI 特性
- **佈局**：`h-screen` 固定比例佈局，內容區具備獨立滾動條。
- **形狀**：大圓角設計 (`rounded-2xl` 至 `rounded-3xl`)。
- **特效**：
  - **Glow Effect**: 邊框微發光與背景擴散光暈。
  - **Animation**: 頁面切換採用 `zoom-in` 或 `slide-in` 動畫。
  - **Line Clamp**: 預覽模式限制文字行數。

## 4. 功能模組 (Functional Modules)

### 4.1 冒險首頁 (Home: Today's Mission)
- **功能**：展示當前進度與指引。
- **視覺元素**：巨大的 "DAY XX" 字樣、任務標題與核心主題。
- **按鈕**：
  - **接取任務**：外部連結至 IT 邦幫忙文章。
  - **前往回報**：切換至佈告欄模式，自動喚起回報表單。

### 4.2 側邊回報面板 (Contextual Sidebar)
- **觸發條件**：僅在「佈告欄」或「酒館」模式顯示。
- **欄位需求**：
  - **勇者稱號**：文字輸入，自動關聯頭像。
  - **外型選擇**：提供 4 個預設頭像種子，具備選中縮放特效。
  - **回報資訊**（僅佈告欄模式）：任務下拉選單、CodePen URL、心得報告。
  - **留言資訊**（僅酒館模式）：長文本輸入框。

### 4.3 任務佈告欄 (Bulletin Board)
- **顯示邏輯**：2x2 網格佈局，每頁固定顯示 4 筆資料。
- **分頁機制**：具備頁碼顯示與左右切換按鈕。
- **鑑定細節 (Modal)**：
  - 點擊「查看完整鑑定」後彈出覆蓋全螢幕的半透明 Modal。
  - 展示未縮略的心得報告、CodePen 連結與完整時間戳記。

### 4.4 公會圖書庫 (Library)
- **功能**：30 天修煉內容存檔。
- **視覺設計**：卡片式書架感。
- **狀態控制**：
  - **Available**: 已解鎖任務，顯示主題並可跳轉。
  - **Locked**: 未開放天數，顯示「封印中」與鎖頭 Icon，具備灰色濾鏡。
- **搜尋**：支援透過標題關鍵字或天數進行即時過濾。

### 4.5 公會酒館 (Tavern)
- **模式**：對話流介面。
- **訊息類型**：
  - **System**: 置中顯示，用於公會公告或勇者踏入提示。
  - **User**: 勇者頭像 + 氣泡對話框。
- **滾動體驗**：自定義極細透明滾動條。

## 5. 資料架構 (Data Structure)

### 5.1 任務物件 (GUILD_TASKS)
```json
{
  "id": "int",
  "day": "string (01-30)",
  "title": "string",
  "theme": "string",
  "link": "url",
  "available": "boolean"
}
```

### 5.2 簽到紀錄 (check_ins)
```json
{
  "id": "timestamp/uuid",
  "name": "string",
  "avatar": "string (seed)",
  "task": "string (Day XX: Title)",
  "url": "url",
  "report": "text",
  "created_at": "datetime"
}
```

### 5.3 酒館訊息 (messages)
```json
{
  "id": "int",
  "user": "string",
  "avatar": "string",
  "content": "text",
  "time": "string (HH:mm)",
  "type": "string (system | user)"
}
```

## 6. 交互流程 (User Flow)
1. **進入**：用戶首見首頁，確認今日 DAY 01 任務內容。
2. **修煉**：點擊「接取任務」至文章頁學習。
3. **回報**：完成 CodePen 後回到 Hub，點擊「前往回報」。
4. **填寫**：左側面板滑入，輸入稱號、選頭像、貼連結、寫心得。
5. **提交**：點擊「傳送回報」，右側佈告欄即時更新，系統自動切換至最新一頁。
6. **查看**：點擊他人卡片，查看完整心得與參考對手代碼。
7. **交流**：切換至酒館，在面板輸入留言，與其他勇者互動。
